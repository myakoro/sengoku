本書は「戦国立身出世SLG」0.3版の唯一の公式要件定義書である。
後続の 1.0 / 2.0 / 3.0 の拡張を妨げない最小構造として設計される。

====================
1. プロジェクト概要
====================
目的：
1. 世界の基本フロー（日次・月次）が破綻なく動くか確認する。
2. 書役としての地味な生活感を、最小の行動モデルで表現する。
3. 将来の拡張を阻害しない最小データ構造を確立する。

0.3版は“ゲーム”ではなく、システム動作確認のためのプロトタイプである。

====================
2. スコープ
====================
以下を 0.3 の実装対象とする：
1. プレイヤーデータ管理（書役）
2. 上司（主君）による評価ロジック
3. 村データ管理（2村）
4. 1日の行動（公務3種＋私事2種）
5. 日次・月次処理
6. 村商人との単純取引

====================
3. 非スコープ（今回扱わないもの）
====================
以下は 0.3 では一切実装しない：
・戦闘（全種類）
・調略
・婚姻
・家臣団
・商人階層（村商人以外）
・世代交代
・村→町→地方の発展
・大名AI
・出世イベント
・自然増減（人口/治安/開発）
・ランダムイベント
・行動失敗判定
・UI要素（画面構成）
・メッセージ/ストーリー要素

====================
4. 世界構造（0.3限定）
====================
構成要素：
1. プレイヤー（書役）
2. 主君（評価更新のみ）
3. 村A（中人口）
4. 村B（小人口）
5. 村商人（各村1名）

====================
5. ゲームサイクル（日・月）
====================
■ 日
・プレイヤーは1日につき1行動を選択。
・行動結果を即時反映。

■ 旬：未使用（将来拡張枠）

■ 月
・月末に以下2つを実行：
　1. 村A/Bの収入を合算し所持金へ加算
　2. 公務を行った月なら評価を増加

これ以外の自動処理は存在しない。

====================
6. プレイヤーデータ仕様
====================
項目：
- name（文字列）
- rank（家格：固定値）
- money（整数：負債許容）
- evaluation（整数：公務によってのみ増加）
- role（書役：固定）

====================
7. 村データ仕様（A/B共通）
====================
項目：
- population（整数：A＞B）
- security（整数）
- income（整数：固定値）
- development（整数）

変動：
・security：公務のみで上昇  
・development：開発でのみ上昇  
・population：変動なし  
・income：固定（変動なし）

====================
8. 主君仕様（評価ロジック）
====================
保持データ：
- name
- evaluation_logic（評価上昇処理）

役割：
・月次処理で、公務実施の有無を確認し評価を増加させる。
・その他の役割は一切持たない。

====================
9. 行動仕様（1日1回）
====================
［公務］
1. 巡察：security + / evaluation +
2. 治安維持：security 大+ / evaluation +
3. 書役仕事：evaluation +

［私事］
4. 開発：development + / money −
5. 取引：money ±（最小処理）

禁止事項：
・変動量は仕様書では決めない（方向のみ）  
・ランダム要素なし  
・失敗判定なし  
・対象村は特に指定がなければ両村を同等に扱う

====================
10. 日次処理の順序
====================
1. 行動選択  
2. データ更新（+ / −）  
3. 公務フラグ更新  
4. 日次終了  
5. 月末なら月次へ遷移

====================
11. 月次処理の順序
====================
1. income(A)+income(B) → money  
2. 公務フラグあり → evaluation +  
3. 公務フラグリセット  
4. 次月へ

====================
