# 戦国立身出世SLG v0.8 UIモック仕様（Windsurf 用）

## 📋 前提条件

- **担当範囲**: UIモック設計とHTML/CSS実装のみ
- **ロジック実装**: 一切行わない（計算・判定・保存処理など）
- **ベースライン**: v0.7 UI を踏襲し、v0.8機能を追加・拡張
- **参照仕様**: 仕様書_v0.8.md（婚姻・縁談・子供・元服・世代交代）

---

## 1. 画面一覧と画面間遷移

### 1.1. 新規追加画面

#### 1.1.1. 婚姻候補一覧画面
**目的**: 他家から届いた縁談オファーを一覧表示し、詳細確認・交渉開始を可能にする

**表示情報**（仕様書 5.1）:
- 相手家名
- 対象者名
- 家格（数値またはランク表示）
- 役職
- 正室/側室区分
- 持参金有無
- 各候補の「詳細を見る」「交渉に進む」ボタン

**アクセス元**:
- メイン画面の「婚姻」ボタン
- メイン画面の「縁談通知バッジ」クリック

#### 1.1.2. 婚姻交渉画面
**目的**: 自家から他家へ縁談を持ちかける際の条件設定と成功率確認

**表示情報**（仕様書 5.2）:
- 相手家情報（家名・家格）
- 対象者情報（名前・役職）
- 現在の成功率（％）
- 提示可能な条件リスト
  - 持参金額候補（スライダーまたはプルダウン）
  - 紹介可能な人脈リスト（チェックボックス）
- 条件選択ごとの成功率変動予測
- 「交渉する」「戻る」ボタン

**アクセス元**:
- 婚姻候補一覧画面の「交渉に進む」ボタン
- 人物詳細画面の「縁談を持ちかける」ボタン（将来的な拡張）

#### 1.1.3. 子供・元服管理画面
**目的**: 家中の子供一覧と元服状況を管理

**表示情報**:
- 子供一覧テーブル
  - 名前
  - 年齢
  - 性別
  - 親（父/母）
  - 成人済み/未成年
  - 元服済みフラグ
  - 後継者フラグ
  - 現在の役職（元服済みの場合）
- フィルター機能（未成年のみ/元服済みのみ）

**アクセス元**:
- メイン画面の「家族」または「子供」ボタン
- 元服通知ポップアップの「詳細を見る」ボタン

#### 1.1.4. 元服通知ポップアップ
**目的**: 子供が15歳に達し元服した際の通知

**表示情報**（仕様書 5.3）:
- 対象者名
- 付与された初期役職名
- 決定要因
  - 「親の威光（基準値）」
  - 「本人の実務（役割）」
  - どちらが採用されたかを明示
- 「確認」「子供一覧を見る」ボタン

**表示タイミング**:
- 月次処理後、元服イベント発生時に自動表示

#### 1.1.5. 世代交代イベント画面
**目的**: 当主死亡時の家督継承情報を詳細に表示

**表示情報**（仕様書 5.4）:
- ヘッダー部
  - 前当主名
  - 新当主名
  - 世代数（例：「第3代当主」）
- パラメータ継承セクション
  - 家格: 100%継承
  - 貸し借り: 60%継承
  - 商人信用: 90%継承
  - 寺社信用: 90%継承
  - 人脈: 100%継承
  - 各項目の変動前→変動後を表示
- 新当主役職セクション
  - 付与された初期役職
  - 決定要因（「親の威光」or「本人の実務」）
- 家中忠誠変化一覧（テーブル）
  - 家臣名
  - 変動前忠誠
  - 変動値（-10 or -20）
  - 変動後忠誠
  - 色分け（成人後継者=-10は黄色、未成年=-20は赤色）
- 「次へ」「ログを閉じる」ボタン

**表示タイミング**:
- 当主死亡判定後、自動表示（ゲーム進行をブロック）

### 1.2. 既存画面への追加要素

#### 1.2.1. メイン画面（日常画面）
**追加要素**:
- **縁談通知バッジ**: 婚姻候補が届いている場合、「婚姻」ボタンに赤丸バッジ表示
- **元服通知アイコン**: 元服可能な子供がいる場合、「家族」ボタンに黄色アイコン
- **世代交代インジケータ**: 当主が高齢（60歳以上）の場合、ヘッダーに「継承準備」アイコン

#### 1.2.2. 月次サマリー画面
**追加要素**:
- **今月の出来事**セクションに以下を追加
  - 「〇〇家から縁談の申し出がありました」
  - 「〇〇が元服しました（役職：△△）」
  - 「子供が生まれました（名前：〇〇）」

## 3. コンポーネント構造

### 3.1. 共通コンポーネント（v0.7から継承）

- `.container`: 画面全体のラッパー
- `.header`: ヘッダー部（タイトル・サブタイトル）
- `.section-title`: セクションタイトル
- `.panel`: 情報パネル
- `.button`: ボタン
- `.badge`: バッジ（通知アイコン）
- `.progress-bar`: プログレスバー

### 3.2. v0.8専用コンポーネント

#### 婚姻関連
- `.marriage-list`: 婚姻候補一覧テーブル
- `.marriage-card`: 婚姻候補カード
- `.marriage-negotiation`: 婚姻交渉画面のラッパー
- `.condition-panel`: 条件設定パネル
- `.success-rate-panel`: 成功率表示パネル
- `.dowry-slider`: 持参金スライダー
- `.connection-checkbox`: 人脈チェックボックス

#### 子供・元服関連
- `.children-list`: 子供一覧テーブル
- `.children-filter`: フィルターボタングループ
- `.genpuku-popup`: 元服通知ポップアップ
- `.heir-badge`: 後継者バッジ（★印）

#### 世代交代関連
- `.succession-screen`: 世代交代画面のラッパー
- `.succession-header`: 前当主→新当主表示
- `.inheritance-panel`: パラメータ継承パネル
- `.loyalty-change-table`: 忠誠変化テーブル
- `.loyalty-decrease-minor`: 忠誠-10（黄色）
- `.loyalty-decrease-major`: 忠誠-20（赤色）

#### 通知・インジケータ
- `.notification-badge`: 縁談通知バッジ（赤丸）
- `.genpuku-indicator`: 元服通知アイコン（黄色）
- `.succession-indicator`: 世代交代インジケータ

## 4. デザイン指針

### 4.1. v0.7との整合性

- **色味**: 茶色(#8b6f47)、ベージュ(#f8f3e7)、金色(#d4a017)、赤色(#c73940)を踏襲
- **フォント**: 游明朝体を継続使用
- **カード配置**: グリッドレイアウトとフレックスボックスの組み合わせ
- **余白**: v0.7と同じpadding/margin値を基準とする

### 4.2. v0.8固有の配色

- **成功率バー**: 緑系グラデーション（高確率）、黄色（中確率）、赤色（低確率）
- **忠誠変化**: 黄色（-10）、赤色（-20）
- **後継者バッジ**: 金色の★印
- **通知バッジ**: 赤丸（縁談）、黄色丸（元服）

### 4.3. レスポンシブ対応

- デスクトップ（1200px以上）: 2〜3カラムレイアウト
- タブレット（768px〜1199px）: 1〜2カラムレイアウト
- モバイル（767px以下）: 1カラムレイアウト

## 5. データ表示の前提

### 5.1. ダミーデータの使用

HTML/CSSサンプルでは、以下のダミーデータをベタ書きする：

- 家名: 田中家、山田家、佐藤家
- 人名: 田中花子、山田美咲、佐藤春香
- 家格: 45〜55の範囲
- 役職: なし、従僕、徒士、組頭
- 持参金: 0〜1000貫
- 成功率: 50〜80%
- 忠誠値: 50〜85

### 5.2. ロジック非実装の明示

- スライダーやチェックボックスは見た目のみ（JSなし）
- 成功率の変動計算は行わない（固定値表示）
- ボタンクリックは遷移先を示すコメントのみ

2. **仕様書準拠**: 表示項目は仕様書_v0.8.mdに記載された内容のみ

4. **追加・拡張のみ**: v0.8機能に必要な画面・要素だけを追加
